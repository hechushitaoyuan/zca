# R2 开发任务书：Docker Hub 多架构流水线与实验部署准备

## 1. 目标

丙方 Claude 负责把当前项目整理成可由乙方快速推送、GitHub Actions 构建并部署到 VPS `/opt/zca` 的最小可用版本。

目标链路：

`main push/tag → E2 tests → linux/amd64 + linux/arm64 build → Docker Hub → /opt/zca compose pull/up → /health 验证`

本轮只做打包、CI、健康/版本标识和部署文档，不改账号、调度、captcha、请求转换或管理 UI 业务逻辑。

## 2. 文件白名单

允许修改/新建：

- `.github/workflows/docker-build.yml`
- `.dockerignore`
- `Dockerfile`
- `compose.yaml`（新建，作为 VPS/生产示例）
- `.env.example`
- `app/settings.py`（仅版本/commit 环境变量）
- `app/routes/pages.py`（仅 `/health` 与 `/meta` 版本信息）
- `tests/test_health.py`（新建）
- `README.md`（仅部署入口与项目名称必要更新）
- `docs/DEPLOYMENT.md`（新建）
- 本轮 R2 交付报告

禁止修改其他源码和 R1 文件。不得读取 `github-Secrets.txt`、`LOCAL-TOKENS.md`、账号包、`.env`、`data/`、私钥或 VPS 生产配置。不得执行 Git 写操作或部署。

## 3. GitHub Actions 要求

1. 触发分支从错误的 `master` 改为当前目标仓库的 `main`；支持 `v*` tag、PR 和手动触发。
2. PR 只测试/构建，不推镜像；`main` push、tag、手动触发可推镜像。
3. Docker Hub 登录只引用：
   - `${{ secrets.DOCKERHUB_USERNAME }}`
   - `${{ secrets.DOCKERHUB_TOKEN }}`
4. 镜像名固定为 `${{ secrets.DOCKERHUB_USERNAME }}/zca`，不得把真实用户名或 token 写入仓库。
5. 平台至少：`linux/amd64,linux/arm64`。
6. 标签至少：默认分支 `latest`、短 SHA、`v*` tag。
7. 构建前必须运行 Python 单测与 `node --check captcha_node/solver.js`；失败不得推镜像。
8. 使用 GitHub Actions cache；permissions 最小化为 `contents: read`。

## 4. 镜像与运行要求

1. Dockerfile 使用 build args 注入版本与 commit，例如 `ZCA_VERSION`、`ZCA_COMMIT`，并暴露为运行时只读环境变量和 OCI labels。
2. 增加不依赖 curl 的 Docker `HEALTHCHECK`，调用容器内 `http://127.0.0.1:3000/health`。
3. `/health` 至少返回：`status=ok`、version、commit；不得返回账号、配置或凭据。
4. `/meta` 同步返回 version/commit。
5. `.dockerignore` 必须明确排除：
   - `github-Secrets.txt`、`LOCAL-TOKENS.md`
   - `IDE-LOG/`
   - `*.zcas.json`、`zcode-accounts.json`、`credentials.json`
   - `.env*`（但镜像不需要 `.env.example`）
   - `data/`、`logs/`、`secrets/`、数据库、key/pem、缓存。
6. 不在镜像内复制测试凭据或运行时数据库。

## 5. compose.yaml 要求

1. service/container 名使用 `zca`。
2. 使用预构建镜像，不在 VPS build：`image: ${ZCA_IMAGE:?set ZCA_IMAGE}`。
3. 映射 `8047:3000`，不得占用或修改 8046。
4. 持久化 `./data:/data`。
5. 使用 `.env` 注入配置；不得硬编码后台密码、网关 Key 或 Docker Hub token。
6. `restart: unless-stopped`，并提供与镜像一致的 healthcheck。
7. `.env.example` 给出 `ZCA_IMAGE=your-dockerhub-user/zca:latest` 等非敏感占位符，并明确后台密码必须修改。

## 6. 部署与回滚文档

`docs/DEPLOYMENT.md` 必须给出：

1. 首次创建 `/opt/zca`、复制 `compose.yaml`/`.env.example` 为 `.env`。
2. `docker compose pull && docker compose up -d`。
3. 检查容器健康、`curl http://127.0.0.1:8047/health`、日志脱敏检查。
4. 通过 SHA tag 固定版本并回滚，不使用删除 volume/数据库的方式回滚。
5. 明确 8046 旧服务不停止、不修改。
6. 私有 Docker Hub 仓库才需要 VPS `docker login`；token 不写入 compose 或文档。

## 7. 自测与交付

必须运行：

```powershell
python -m compileall -q app tests main.py
python -m unittest discover -s tests -v
node --check captcha_node/solver.js
```

若 Docker daemon 可用，再运行本机单平台 build；不可用则明确 `BLOCKED`，由 GitHub Actions 完成多架构 E2 验证。

交付报告命名：

`YYMMDDHHMM-R2-汇报-DockerHub流水线与部署准备交付.md`

报告包含实际文件、workflow 触发/权限/标签矩阵、测试命令退出码、Docker 可用性、残余风险和文件锁释放声明。不得执行 Git 写操作。
