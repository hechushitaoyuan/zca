## 2. 逐文件改动目的

### 2.1 `captcha_node/solver.js`

- **基线**：以 PR #5 头提交 `aac09c7`（与 `525ff04` 内容一致，已 `git show` 核对）为技术基线，**替换掉当前 HEAD 中缺失指纹 polyfill 的简化实现**（即任务书所指 exit code 4 版本）。
- **保留基线全部能力**：完整 navigator/screen/canvas/webgl/Worker/OffscreenCanvas 指纹 polyfill、`AliyunCaptchaConfig`、`language:'en'`、固定 UA。
- **scene 默认 `11xygtvd`**，region/prefix 由 `process.argv[3]/[4]` 传入并保留安全默认值（`sgp`/`no8xfe`）。
- **成功**：仅通过 `console.log('VERIFY_PARAM=' + param)` 输出，随后 `process.exit(0)`。
- **退出码**：success→0、timeout(25s)→2、顶层 catch→3、fail→4、onError→5（与基线一致，可区分）。
- **唯一相对基线的改动 = 脱敏**：新增 `diag(tag, detail)` 统一处理 stderr，所有诊断（jsdom error/warn、start、fail、onError、fatal）压成单行并限长 200 字符。基线原本 `process.stderr.write('fail=' + JSON.stringify(err))` 无长度上限、可能回显完整错误体，已全部改为短标签输出。**普通日志不打印 verifyParam。**

### 2.2 `app/captcha.py`

受控子进程与缓存重写：

- **可配置超时 + 完整回收**：`asyncio.wait_for(proc.communicate(), timeout=CAPTCHA_SOLVE_TIMEOUT)`，超时后 `_terminate()` 执行 `kill()` 再 `await proc.wait()`，避免僵尸进程。
- **可区分异常**：`SolverError` 基类，派生 `SolverTimeout`/`SolverExitError`/`SolverOutputError`，分别对应超时、非零退出、（空输出/缺标记/过短）。
- **最小长度校验**：`MIN_VERIFY_PARAM_LEN = 32`，过短判失败且**只透出长度 `len=N`，不回显参数原值**。
- **脱敏限长**：`_tail()` 把 stderr/stdout 诊断压单行、限长 200，异常信息不含完整 JWT/Key/verifyParam。
- **TTL 缓存 + single-flight**：`asyncio.Lock` + 入锁二次检查，TTL（45s，沿用 `CAPTCHA_CACHE_TTL`）内复用，并发只跑一次 solver。
- **region 传递**：`get_verify_param()` 返回 `(param, region)`，region 来自 `fetch_config()` 的配置，不再永远硬编码 `sgp`。
- **invalidate()**：清空缓存，下次强制重新求解。
- **可测缝隙**：`_create_subprocess(argv)` 独立成方法，测试可替换以计数 / 捕获 argv / 注入假进程。

> 说明：`MIN_VERIFY_PARAM_LEN`、`DIAG_MAX_LEN` 等新常量放在 `captcha.py` 模块内，**未改 `settings.py`**（该文件为只读参考）；TTL/超时/重试次数沿用 settings 既有常量。

## 4. ARM64 与镜像评估

- `package-lock.json` 中出现的 `canvas`、`bufferutil`、`utf-8-validate` 均为 **optional peerDependency**（`peerDependenciesMeta.*.optional = true`），lock 中**无对应 `node_modules/` 实际安装条目**，`npm ci --omit=dev` 不会编译它们。
- jsdom 纯 JS，无 amd64-only 原生依赖。
- 结论：**当前 Dockerfile 已满足 `linux/amd64` + `linux/arm64` 可构建条件，按任务书 4.3.3 不为"看起来更完整"而改动**。

## 7. 残余风险

1. **P2｜真实求解未验证**：solver.js 指纹 polyfill 移植自"已验证"基线，但本轮仅做语法检查，未在真实网络下验证能否产出可通过的 verifyParam。能否真正绕过当前阿里云风控属 E3，需乙方/甲方授权后低频实测。
2. **P2｜`MIN_VERIFY_PARAM_LEN=32` 为保守经验值**：用于挡空/截断输出。若真实 verifyParam 短于 32（可能性低），会误判失败。建议乙方在 E3 实测时核对真实参数长度后确认阈值。
3. **超时默认 40s（`CAPTCHA_SOLVE_TIMEOUT`）> solver 内部 25s 兜底**：solver 自身 25s 会 `exit(2)`，Python 端 40s 是二级保险，行为符合预期，但两值耦合在 settings 与 solver 两处，调整时需同步注意。
