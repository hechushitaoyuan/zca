# R3 指导：安全公网访问与网关密钥初始化

**角色**：乙方 Codex → 丙方 Claude
**背景**：甲方要求通过 `http://132.226.70.137:8047/admin` 访问实验服务。当前 compose 为安全起见固定绑定 `127.0.0.1`，且已有数据库的 `gateway_key` 为空，不能直接长期公开。

## 目标

1. 支持通过环境变量为首次部署及“数据库值仍为空”的既有部署安全初始化网关 API Key。
2. 允许部署者显式选择监听地址，默认仍保持回环安全值。
3. 不覆盖数据库中已经由用户设置的非空网关 Key。

## 文件白名单

- `app/settings.py`
- `app/store.py`
- `compose.yaml`
- `.env.example`
- `docs/DEPLOYMENT.md`
- `tests/` 下本任务新增或修改的测试
- 本任务交付报告

## 实现要求

- 新增 `ZCODE_GATEWAY_KEY` 环境变量，默认空。
- 新数据库用该值初始化 `gateway_key`。
- 既有数据库仅在当前 `gateway_key` 为空且环境变量非空时写入；不得覆盖非空用户配置。
- compose 端口使用 `${ZCA_BIND_IP:-127.0.0.1}:8047:3000`；`.env.example` 解释 `127.0.0.1` 与 `0.0.0.0` 的差异。
- 文档明确：设置 `ZCA_BIND_IP=0.0.0.0` 前必须配置非空强 `ZCODE_ADMIN_KEY` 与 `ZCODE_GATEWAY_KEY`，并确认云防火墙范围。
- 日志、测试输出和报告不得回显真实 Key。

## 测试要求

- 新数据库正确使用环境网关 Key。
- 既有空值可以由环境值安全补齐。
- 既有非空值绝不被环境变量覆盖。
- 未配置环境值时保持现有兼容行为。
- 复跑全部 unittest、compileall、node check、diff check；不发真实上游请求，不做 Git 写操作。
